Representaties

Elke laag heeft zijn eigen representatie van de data.

  • ApiController: ApiContract ~ DTO

  • Service: DomainModel ~ Model

  • Persistence: DataModel ~ DAO

Dit wil zeggen dat we mapping tussen de lagen moeten voorzien. Er is veel vrijheid in de manier waarop deze projecten elkaar zouden kunnen kennen.

  • Waar worden interfaces en implementaties aangemaakt?

  • Welke laag zal wat naar wat mappen?

Een optie: Repo gebruikt intern data model, maar haar publieke methodes werken met domein model
→ Mapping gebeurt in Persistence

Andere optie: Repo gebruikt intern data model, haar publieke methodes werken met diezelfde data model klassen
→ Mapping gebeurt in Service

etc…​ Er zijn nog veel varianten mogelijk.

Ook kunnen deze lagen eerst als folders binnen één project gemaakt worden, en pas later opgesplitst worden in aparte projecten. Hier is geen eenduidig juist of fout antwoord voor.

Als je zeker weet hoe je wil werken, werk je uiteraard vanaf het begin in die richting.
