RESTful Api

Doel

Bouwen van een .Net backend api voor een Storefront applicatie. We stellen als initieel doel het beheer van

  • Klanten

  • Producten

  • Bestellingen

Terminologie

We nemen ons voor de oplossing in het Engels te programmeren. We kiezen dus een passende Engelstalige term voor de concepten.

There are only two hard things in Computer Science: cache invalidation and naming things.
— Phil Karlton
Klant ⇒ Customer

Komende van het Nederlands, klinkt Client als een goede keuze, maar dat is het niet. Client wordt gebruikt in vele contexten: Client-Server, Client Credentials, HttpClient…​ Customer is meestal de betere keuze.

Product ⇒ Product

Product is een prima keuze, tenzij we ook diensten gaan verkopen. Gelukkig gaan we dat niet doen.

Bestelling ⇒ Order

Er bestaat een semantisch verschil tussen aankoop (purchase) en bestelling (order). Een aankoop wordt onmiddellijk voltrokken en een bestelling impliceert een wachttijd.
Stel: je wandelt een elektronicazaak binnen met de intentie een bepaalde laptop aan te kopen. Wanneer ze deze niet in voorraad hebben, zal je een bestelling kunnen plaatsen.
Gezien er hedendaags online ook digitale producten verkocht worden die wel direct geleverd worden, vervaagt de grens tussen aankoop en bestelling dat we ze in de praktijk door elkaar kunnen gebruiken. Order lijkt de meest gebruikte optie.

Kort door de bocht kunnen we stellen dat we POST, GET, PUT en DELETE ondersteuning willen op /customers, /products en /orders.

Uiteraard gaan we dit moeten uitdiepen. Wat is een Customer? Welke response codes verwachten we? …​

Customer

Model

Uit grondige analyse besluiten we dat een Customer bestaat uit

  • FirstName

  • LastName

  • DateOfBirth

  • Email

  • TaxIdentificationNumber (Nullable)

  • AddressLine1

  • AddressLine2

  • AddressLine3 (Nullable)

  • Country

Api

POST (Create)
POST api/customers
    Expected Request
        Body: Customer Object
    Expected Response
        Status: 201 - Created
        Body: Empty
GET (Read)
GET api/customers
    Expected Request
        Body: Empty
    Expected Response
        Status: 200 - Ok
        Body: Customer Object []

GET api/customers/{id}
    Expected Request
        Body: Empty
        Path: {id} = Valid Customer Id
    Expected Response
        Status: 200 - Ok
        Body: Customer Object
PUT (Update)
PUT api/customers/{id}
    Expected Request
        Body: Customer Object
        Path: {id} = Customer Id
    Expected Response
        Status: 200 - Ok
        Body: Empty
Delete (Delete)
DELETE api/customers/{id}
    Expected Request
        Body: Empty
        Path: {id} = Valid Customer Id
    Expected Response
        Status: 200 - Ok
        Body: Empty
