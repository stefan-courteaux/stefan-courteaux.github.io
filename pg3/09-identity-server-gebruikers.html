IS UI Toevoegen

Identity Server biedt een implementatie van OpenID Connect die we kunnen gebruiken om user auth te doen.

Om mensen door de login flow te loodsen hebben we schermen nodig.Je kan deze volledig zelf ontwikkelen of vertrekken van de Identity Server templates.

We gebruiken de ingebouwde ui-template van Identity Server. Indien gewenst, kan je deze restylen.

src\ShipIt\ShipIt.IdentityServer
dotnet new isui

Het is mogelijk dat nuget Duende.IdentityModel. Voeg deze toe en fix de refernces indien nodig.

De nodige calls om de user interface te activeren staan al in de code, maar moeten uit commentaar gezet worden.

ShipIt.IdentityServer\HostingExtensions.cs
public static WebApplication ConfigureServices(
    this WebApplicationBuilder builder)
{
    // uncomment if you want to add a UI
    builder.Services.AddRazorPages();
    ...
}

public static WebApplication ConfigurePipeline(
    this WebApplication app)
{
    ...
	// uncomment if you want to add a UI
	app.UseStaticFiles();
	app.UseRouting();
	...
	// uncomment if you want to add a UI
	app.UseAuthorization();
	app.MapRazorPages().RequireAuthorization();
	...
}

Testen

Start het IdentityServer project op en surf naar https://localhost:5001.

Klik op "Click here to see the claims of your current session."

Log in met een van de test users bob/bob of alice/alice.

De web pagina zal de informatie uit het huidige token weergeven.

Bovenaan in de navigatiebalk staat een menu item dat dient als bevestiging dat je ingelogd bent. Je kan daar ook uitloggen.

De user interface van Identity Server is nu operationeel. We kunnen deze users gebruiken in een eigen front-end applicatie.
