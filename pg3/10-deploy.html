App Deployen

Build

Build Solution
dotnet build ShipIt.sln

Settings

Zorg ervoor dat alle urls naar andere services uit config gelezen worden, geen hardcoded waarden.

Zet alle appsettings.json waarden juist voor Azure omgeving.

  • Identity Server,

  • Quote Api,

  • Externe Open Data Api’s,

  • Databases

  • Client keys en redirect urls

  • …​

Bedenk of je de Identity Server database moet hersyncen met in-memory configuratie.

Deploy Services

Zoals altijd, backslash voor Linux/macOS en backtick voor Windows als escape character. Indien je er problemen mee hebt, zet alles gewoon op 1 lijn.
IdentityServer
az webapp up \
  --name hogent-pg3-stefanc-shipit-identity \
  --plan hogent-pg3-stefanc-shipit-plan \
  --resource-group hogent-pg3-stefanc-shipit \
  --location canadacentral \
  --os-type linux
--> https://hogent-pg3-stefanc-shipit-identity.azurewebsites.net

Activeer in de Azure Portal de mogelijkheid om een website te hosten van op een storage account. Dit zal een $web container maken.

De Primary Endpoint url in het screenshot hierboven is het publieke adres van jouw index.html!

Activeer de mogelijkheid om bestanden publiek leesbaar te maken. Dit wil niet zeggen dat alle bestanden plots publiek staan. Het wil zeggen dat je bestanden kan markeren als publiek.

Zet de $web container publiek

Navigeer naar jouw web project in terminal en upload de build.

Push Web App Build Output
az storage blob upload-batch \
  --account-name hogentpg3stefancshipit \
  --source ./dist \
  --destination '$web' \
  --overwrite

UI Test

Kopieer de Primary Endpoint url uit Azure Portal en navigeer met web browser. In mijn geval is dit.

https://hogentpg3stefancshipit.z9.web.core.windows.net/

Hoogst waarschijnlijk krijg je CORS errors in de browser.

In het geval van identity server moet je op de front end app client settings de origin toevoegen waarop de site gehost wordt. Ook hier goed beseffen of je config uit geheugen gelezen wordt (Config.cs) of de SQL database.

Config.cs
//...
new Client {
    ClientId = "react-spa-client",
    ClientSecrets = {new Secret("react-secret".Sha256())},
    AllowedGrantTypes = GrantTypes.Code,
    //...
    RedirectUris = { "http://localhost:5173/",
        "https://hogentpg3stefancshipit.z9.web.core.windows.net/"},(1)
    PostLogoutRedirectUris = { "http://localhost:5173/",
        "https://hogentpg3stefancshipit.z9.web.core.windows.net/"},(1)
    AllowedCorsOrigins = ["http://localhost:5173",
        "https://hogentpg3stefancshipit.z9.web.core.windows.net"],(1)
}
//...
1 Hier gebeurt het.

In het geval van de eigen backend api’s doe je dit in de Program.cs of via de CORS settings in Azure Portal.

Als alle CORS settings juist staan, zal je een werkende site en cleane log (aka geen errors) in de browser hebben.
